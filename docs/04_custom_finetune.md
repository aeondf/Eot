# Вариант 3: Файнтюн своей модели

## Когда это нужно

Если варианты 1 и 2 не дают нужного качества на наших данных. Колл-центр — специфичный домен: много пауз при зачитывании данных, перечислений, "угу / ага" посередине.

## Идея

Берём готовую предобученную модель и файнтюним под наш домен и язык.

## Два направления

### А. Текстовый классификатор (проще)

Берём ruBERT или XLM-RoBERTa, файнтюним на транскриптах.

```
текст → ruBERT → [CLS] → linear head → complete / incomplete
```

**Данные**: берём публичные русскоязычные диалоговые датасеты (OpenSubtitles RU, RuDialogues) + синтетика.  
Генерация incomplete: обрезаем полные реплики на случайном токене.

Минус: зависим от ASR, не ловим просодику.

### Б. Аудио-классификатор (точнее, дольше)

Берём Pipecat Smart Turn v3 и файнтюним на русских аудио из колл-центра.

```
PCM16 → Whisper Tiny (frozen encoder) → linear head (обучаем) → complete / incomplete
```

**Данные**: нужны записи звонков с разметкой (где кончается реплика). 1000-5000 примеров — достаточно для файнтюна.

Плюс: ловит интонацию, не зависит от ASR.

## Что нужно для файнтюна

- Датасет размеченных реплик (complete/incomplete) — **пока нет**
- Можно сгенерировать синтетически через TTS на русском
- GPU для обучения (есть)
- 1-3 дня на подготовку данных + обучение

## Метрики оценки

- F1 на held-out сете
- FP rate: отправили в LLM, а пользователь не закончил
- FN rate: не отправили, хотя закончил (latency растёт)
- Latency инференса

## Когда делаем

Только если варианты 1 и 2 показали плохое качество на наших тестовых примерах. Не раньше.
