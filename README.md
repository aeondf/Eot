# Turn Detection — спайк

Как понять, что пользователь **закончил реплику** в голосовом звонке, и пора
отправлять текст в LLM. Сейчас это решается просто по тишине (N секунд), но это
либо медленно, либо обрывает человека на паузе. Спайк проверяет, можно ли
добавить умную модель, которая по голосу/тексту предсказывает завершённость.

## Где это в пайплайне

```
PCM16 (WS) → VAD (речь/тишина) → ASR (Qwen) → текст
                   │                              │
                   └──────→ turn-детектор ←───────┘
                                  │ score 0..1
                                  ▼
        (score > порог И тишина > 1с) ИЛИ тишина > 5с → LLM
```

Turn-детектор — **слой поверх VAD**


## Что сравнивали

| | Smart Turn v3 | LiveKit turn-detector |
|---|---|---|
| Вход | аудио (PCM 16kHz) | текст (транскрипт) |
| Зависит от ASR | **нет** (идёт параллельно) | да (ждёт транскрипт) |
| Модель | Whisper Tiny + head, ~8M | Qwen2.5-0.5B, ~500M |
| Размер (int8) | ~8 MB | ~500 MB |
| Инференс CPU | ~12–22 ms | ~10 ms (растёт с длиной текста) |
| Языки | 23 (вкл. рус, укр) | 14 (вкл. рус) |

## Результаты спайка

### Smart Turn v3 (аудио) 

Прогон на **официальном** тест-сете `pipecat-ai/smart-turn-data-v3.1-test`,
русская часть (165 семплов, реальные метки, 87 complete / 78 incomplete):

- **ROC-AUC = 0.969** — 
- Средний score: complete **0.944**, incomplete **0.181** (разрыв огромный)
- При пороге **0.7**: accuracy 0.92, precision 0.89, recall 0.97
- Latency ~22 ms на CPU (есть GPU-версия модели)


### LiveKit turn-detector (текст)

Прогон на 3 датасетах (метки через обрезку: полная фраза = complete,
обрезанная в середине = incomplete). Главная находка — **домен-чувствительность**:

| Датасет | Домен | ROC-AUC |
|---|---|---|
| `d0rj/dialogsum-ru` | разговорные бытовые диалоги (≈колл-центр) | **0.879** |
| `inkoziev/Conversations` | книжные диалоги | 0.81 |
| smart-turn (транскрипт Whisper) | начитка / энциклопедия | 0.43 |

Чем ближе текст к живому диалогу — тем лучше. На начитанном повествовании
модель проваливается (хуже случайного): длинную фразу из книги она не считает
«завершённым ходом диалога». Наш колл-центр разговорный → ориентир **~0.88**.

На разговорном домене (dialogsum-ru):
- **ROC-AUC = 0.879**, avg score: complete **0.29**, incomplete **0.06**
- При пороге **0.2**: precision **0.84**, recall 0.64, FPR 0.12

Оговорки:

1. **Масштаб score низкий.** `score > 0.9` для LiveKit не сработает никогда —
   рабочий диапазон в районе сотых-десятых. Порог подбирается под модель.
2. **Метки через обрезку шумят** (часть «incomplete» на деле завершена) →
   реальная точность ещё выше измеренной.
3. Сравнение со Smart Turn не идеально честное: разные данные и метки. Честно
   рассудят только реальные размеченные звонки.

## Вывод

**Smart Turn v3** Легче по памяти (8 МБ против 500 МБ),
не ждёт ASR (быстрее end-to-end), отлично разделяет классы на реальном аудио
(AUC 0.969). Наш формат PCM16 — на вход напрямую.

**LiveKit** Понимает текст, ловит смысловую
незавершённость (диктовка номеров, перечисления), которую аудио-модель может
пропустить. 

### Рекомендуемые пороги (стартовые, переподобрать)

| Smart Turn v3 | **0.7** | precision 0.89, recall 0.97 (реальные метки) |
| LiveKit | **0.20** | precision 0.84, recall 0.64 (разговорный домен) |
